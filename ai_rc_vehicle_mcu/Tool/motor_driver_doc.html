2.5 AI와 함께하는 TC237 모터 구동 실습

이번 절에서는 실제로 Claude Code에 요청하여 TC237 기반 4WD 모터 드라이버를 설계합니다. AI가 생성한 코드를 그대로 쓰는 것이 아니라 — 어떻게 요청하고, 어떻게 검증하고, 어떻게 수정하는지를 함께 살펴봅니다.

AI에게 드라이버 코드 요청하기

Claude Code에 다음과 같이 요청합니다.

TC237(AURIX)에서 L298N 모터 드라이버로 DC 모터 4개(4WD)를 제어하는 코드를 작성해줘. ILLD 라이브러리 사용, 다음 구조로: [하드웨어 핀] PWM (GTM TOM0, 1kHz): FL: TOM0_Ch0 → P33.4 FR: TOM0_Ch1 → P33.5 RL: TOM0_Ch6 → P02.4 RR: TOM0_Ch13 → P02.5 방향 제어 GPIO (H-Bridge IN1/IN2): FL: IN1=P33.2, IN2=P33.3 FR: IN1=P33.1, IN2=P33.12 (방향 반전 보정) RL: IN1=P00.0, IN2=P00.1 (방향 반전 보정) RR: IN1=P00.2, IN2=P00.3 [계층 구조] HAL Layer : DrvPwm.c / DrvDio.c (ILLD 직접 호출) Driver Layer: DrvMotor.c (HAL 래퍼) App Layer : AppVehicle.c (차량 동작 로직) [DrvMotor API] void DrvMotor_Init(void); void DrvMotor_SetDuty(MotorId id, sint16 duty); // -100~+100, 음수=역회전 void DrvMotor_Brake(MotorId id); // 능동 제동: IN1=H, IN2=H void DrvMotor_Coast(MotorId id); // 자유 회전: IN1=L, IN2=L, PWM=0 방향 전환 시 슛스루 방지 시퀀스: PWM=0 → GPIO 방향 변경 → PWM 설정 순서로. [AppVehicle] 명령: STOP / FORWARD / REVERSE / SPIN_LEFT / SPIN_RIGHT 전륜 duty = speed * 0.86 (언더스티어 보정) 초기 속도 80%

드라이버 계층 설계

[Application Layer]   ← RC카 동작 로직
  AppVehicle_Update()
       |
       ▼
[Driver Layer]        ← L298N 추상화
  DrvMotor_SetDuty(id, duty)   // -100 ~ +100
  DrvMotor_Brake(id)
  DrvMotor_Coast(id)
       |
       ▼
[HAL Layer]           ← TC237 ILLD 직접 호출
  IfxGtm_Tom_Timer_setTrigger()      // PWM 듀티
  IfxPort_setPinHigh / setPinLow()   // IN1, IN2 제어
설계 원칙: Driver Layer는 L298N을 모른다. HAL Layer만 하드웨어를 안다. Application Layer는 방향이나 PWM을 모른다. duty 부호로 모든 것을 표현한다.

생성된 API 명세

DrvMotor.h — Driver Layer

typedef enum
{
    MOTOR_FL    = 0,   /* Front-Left  */
    MOTOR_FR    = 1,   /* Front-Right */
    MOTOR_RL    = 2,   /* Rear-Left   */
    MOTOR_RR    = 3,   /* Rear-Right  */
    MOTOR_COUNT = 4
} MotorId;

void DrvMotor_Init(void);
void DrvMotor_SetDuty(MotorId id, sint16 duty);  /* -100 ~ +100 */
void DrvMotor_Brake(MotorId id);
void DrvMotor_Coast(MotorId id);
함수설명
DrvMotor_Init()내부 방향 상태 초기화. DrvPwm_Init(), DrvDio_Init() 이후 호출.
DrvMotor_SetDuty(id, duty) duty > 0 : 정회전, duty < 0 : 역회전, duty = 0 : 정지.
방향 전환 시 자동으로 PWM=0 → 방향 변경 → PWM 설정 순서 실행.
DrvMotor_Brake(id)능동 제동. IN1=H, IN2=H 상태로 모터를 단락시켜 즉시 제동.
DrvMotor_Coast(id)자유 회전. PWM=0, IN1=L, IN2=L. 모터가 관성으로 멈춤.

방향 전환 슛스루 방지 시퀀스

AI 코드를 그대로 믿지 마세요.
방향 전환 시 PWM을 먼저 0으로 내리지 않으면 — H-Bridge 양쪽 스위치가 동시에 켜지는 슛스루(Shoot-through) 현상이 발생할 수 있습니다. AI가 생성한 코드에서 이 부분이 빠지는 경우가 종종 있습니다.

AppVehicle — Application Layer

명령FLFRRLRR
VEHICLE_FORWARD+spdF+spdF+spd+spd
VEHICLE_REVERSE-spdF-spdF-spd-spd
VEHICLE_SPIN_LEFT-spdF+spdF-spd+spd
VEHICLE_SPIN_RIGHT+spdF-spdF+spd-spd
VEHICLE_STOPCoast (전 모터)

spdF = speed × 0.86 (전륜 언더스티어 보정)

동작 확인 — 테스트 시퀀스

코드를 빌드하고 보드에 다운로드한 뒤, RC카를 평평한 바닥에 놓고 다음 시퀀스로 테스트합니다.

/* AppMotorTest.c — 동작 확인 시퀀스 */

void AppMotorTest_Sequence(void)
{
    /* 전진 2초 */
    DrvMotor_SetDuty(MOTOR_FL, +50);
    DrvMotor_SetDuty(MOTOR_FR, +50);
    DrvMotor_SetDuty(MOTOR_RL, +50);
    DrvMotor_SetDuty(MOTOR_RR, +50);
    delay_ms(2000);

    /* 역회전 2초 */
    DrvMotor_SetDuty(MOTOR_FL, -50);
    DrvMotor_SetDuty(MOTOR_FR, -50);
    DrvMotor_SetDuty(MOTOR_RL, -50);
    DrvMotor_SetDuty(MOTOR_RR, -50);
    delay_ms(2000);

    /* 능동 제동 2초 */
    DrvMotor_Brake(MOTOR_FL);
    DrvMotor_Brake(MOTOR_FR);
    DrvMotor_Brake(MOTOR_RL);
    DrvMotor_Brake(MOTOR_RR);
    delay_ms(2000);

    /* 정지 */
    DrvMotor_Coast(MOTOR_FL);
    DrvMotor_Coast(MOTOR_FR);
    DrvMotor_Coast(MOTOR_RL);
    DrvMotor_Coast(MOTOR_RR);
}

RC카가 — 2초 전진, 2초 후진, 2초 정지를 반복하면 성공입니다.

만약 한쪽 모터가 반대 방향으로 도는 것 같다면 — 그 모터의 OUT1/OUT2 단자를 서로 바꿔 끼우면 됩니다. 코드를 수정할 필요는 없습니다.
PWM 주파수에 대하여
20kHz는 가청 주파수를 피하기 위한 선택이지만, L298N은 구형 바이폴라 IC라 고주파 스위칭 시 내부 손실이 커집니다. 실측 결과 1kHz에서 토크가 더 강하게 나타났습니다. L298N 환경에서는 1kHz가 최적입니다.
